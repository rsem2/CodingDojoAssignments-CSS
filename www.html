Clients and Servers

Computers connected to the Web are called clients and servers.

  • Clients are the typical Web user's internet-interconnected devices (for exampls. yourr computer connected to you Wi-Fi, or your phone connected yo your mobile metwork) and Web-accessing software available on those devices (usually a web browser like Firefox or Chrome).
  • Servers are computers that store webpages, sites, or apps. When a client device wants to access a webpage, a copy of the webpage is downloaded from the server onto the client machine to be displayed in the user's web browser.
So What Actually Happens?
  1. The browser goes down to the DNS server and finds the real adderss of the server that the website lives on.
  2. The browser sends an HTTP request message to the server asking it to sent a copy of the wensite to the client.
  3. If the server approves the client's request, the server sends the slient a "200 OK" message, which means "Permission granted! Her you go!", and then starts sneding the website's files tot he browser as a series of small chunks.
  4. The browser assembles the small chunks into a complete website and displays it to you!
